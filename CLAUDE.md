# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Что это

Хакатонский прототип по кейсу **«Полёт вслепую»**: определение координат, скорости и направления
движения воздушного судна **без ГНСС**, в районах, где спутниковая навигация отказывает (тайга, тундра,
горы, высокие широты). Метод — **TERCOM** (Terrain Contour Matching, корреляция рельефа): живой профиль
высот рельефа, полученный с радиовысотомера, сопоставляется с цифровой моделью рельефа (DEM), чтобы
определить положение на карте.

Всё решение — два самостоятельных Python-скрипта. Нет пакета, системы сборки и тестов.

## Команды

Файла `requirements.txt` нет. Зависимости ставятся вручную:

```bash
pip install numpy scipy matplotlib       # основное (без scipy/matplotlib скрипт работает в урезанном режиме)
pip install rasterio tifffile            # опционально, только для DEM в формате GeoTIFF
```

Опциональный шаг подготовки — конвертация сырого NMEA-лога в текстовый файл «одна высота на строку»:

```bash
python convert_nmea_to_heights.py --nmea nmea_data_sibir.txt --out heights_m.txt
```

Запуск навигатора (полностью пересоздаёт содержимое `result/`):

```bash
python tercom_final.py \
  --dem dem_package_sibir.txt \
  --heights heights_m.txt \
  --truth tchk.csv \
  --out-dir result
```

- `--truth tchk.csv` — **только для отладки**: задаёт стартовую точку и включает метрики ошибки и графики.
  В реальном запуске «без GPS» вместо него используются `--start-x/--start-y` (или `--start-lon/--start-lat`)
  плюс `--initial-heading-deg`. GPS, если он есть, используется *только* как последняя известная стартовая точка.
- `--nmea <файл>` — устаревшая альтернатива `--heights`; если заданы оба, побеждает `--heights`.
- Демо/точный режим 1°: добавить `--bootstrap-az-step-deg 1 --heatmap-az-step-deg 1` (медленнее).
- `--profile` пишет тайминги в `result/profile_summary.json`. `--no-plot-heatmap` отключает тепловую карту.

У `tercom_final.py` около 80 флагов CLI (`build_parser()` в конце файла — источник истины для настройки
матчера, фильтра Калмана и предобработки сенсоров). Значения по умолчанию подобраны под вложенные данные.

## Архитектура

### Поток данных
1. **Радиовысотомер** NMEA `$GPGGA`, поле 9 → отсчёты высоты (`heights_m.txt`, 10 Гц).
2. **Модель сенсоров** (`make_terrain_profile` / ветка `--heights` внутри `run`): радиоканал проходит
   `median_filter` + фильтр Баттерворта `lowpass`; синтетический барометр (константа `--absolute-altitude`
   + опциональный шум/дрейф) фильтруется `median_filter` + `BaroKalman1D`. **terrain = баро − радио**.
   Это и есть живой профиль, который сопоставляется с DEM.
3. **Сопоставление TERCOM** с DEM, результат подаётся в мультигипотезный трекер Калмана.
4. **RTS-сглаживание траектории** (`rts_smooth_positions`, флаг `--smooth`, включён по умолчанию):
   офлайн-сглаживатель Рауха–Тунга–Штрибеля (прямой+обратный проход) по модели постоянной скорости.
   Шум измерения на каждом сэмпле масштабируется обратно `solution_confidence`, поэтому кратковременные
   уходы трекера на ложную гипотезу (низкая уверенность) «перекрываются» динамикой и уверенными соседями,
   а реальные манёвры (высокая уверенность) сохраняются. Сырой выход трекера остаётся в колонках
   `x_raw`/`y_raw`. Честно (использует только собственную уверенность, не истину); на этих данных убрал
   переходный выброс ~28 px и снизил mean/p95 примерно вдвое.
5. **Доводка траектории по корреляции** (`correlation_refit`, флаг `--refit`, выключена по умолчанию —
   медленнее на ~1 мин): на подвыборке точек (`--refit-step`) детерминированно снимает пик корреляции
   рельефа рядом с текущей оценкой (ограниченный поиск ±`--refit-radius-px`), отвергает поправки больше
   `--refit-gate-px` как ложные пики на неоднозначном рельефе, и пере-сглаживает через принятые якоря
   (`_rts_core`). Без обратной связи в MHT. На этих данных снизил медианную ошибку с ~32 м до ~16 м
   (близко к физическому потолку ~11 м). Честно — без истины.
6. **Финальная коррекция позиции** (`final_position_fix`, флаг `--final-fix`, включён по умолчанию):
   детерминированный плотный поиск пика корреляции в окрестности сошедшейся оценки, усреднённый по
   нескольким длинам окна с весом по корреляции. **Без обратной связи в MHT** — поэтому, в отличие от
   уточнения на каждом измерении, не «перещёлкивает» гипотезы и не дестабилизирует трек. Снижает
   финальную ошибку на этих данных с ~13 м до ~4 м (порядка размера пикселя DEM — это честный предел;
   пик корреляции лежит в ~6–10 м от истины, ниже не опуститься без вранья). Истина (`--truth`) при этом
   не используется.
7. **Выходные файлы** в `--out-dir`: `estimated_trajectory.csv` (полное состояние по каждому отсчёту),
   `trajectory_local.csv` (пиксели), `trajectory_global.csv` (lon/lat/скорость/курс),
   `correlation_events.csv` (решения матчера), `report.html` и PNG-графики.

### Сопоставление, две фазы (главный цикл в `run()`)
- **Bootstrap** (`bootstrap_match`, пока нет захвата): скорость и курс неизвестны, поэтому перебирается
  сетка азимут × скорость × малые смещения (dx,dy) от последней точки GPS, и для каждого кандидата
  коррелируется профиль вдоль линии (`extract_line_profile` + `corr_rows`). Многомасштабная проверка по
  нескольким длинам окна (`--bootstrap-verify-windows`) отсекает ложные захваты (в частности классический
  «развёрнутый на 90°» обман).
- **После захвата**, два режима через `--local-match-mode`:
  - `segment` (`segment_match`): короткий прямой отрезок от якоря недавней истории, ограниченный окрестностью
    предсказанной Калманом конечной точки — устойчив к манёврам.
  - `history` (`history_match`, по умолчанию): сдвиг/поворот предсказанного профиля истории — лучше для
    плавного/прямолинейного полёта.

### Трекинг и устойчивость
- **Мультигипотезный трекинг (MHT)**: список `Hyp`, каждый — фильтр постоянной скорости `KalmanCV`
  `[x,y,vx,vy]` плюс score/история. На каждом цикле кандидаты расширяются, переоцениваются,
  дедуплицируются через NMS (`top_n_candidates`), остаются лучшие `--max-hypotheses`. `clone_hyp` ветвит
  гипотезу под каждого принятого кандидата.
- **Отсев плоского рельефа**: `roughness()` гейтит сопоставление; безликий рельеф пропускается/штрафуется,
  так как по нему нельзя локализоваться. `solution_confidence()` объединяет качество корреляции и шероховатость.
- `nav_status` по каждому отсчёту — `LOCKED` или `DEAD_RECKONING_NO_GPS`.

### Координаты
- **Внутренние единицы — пиксели DEM: `x = столбец (col)`, `y = строка (row)`.** Перевод пиксели↔география
  через GDAL-геотрансформ DEM (`_gdal_coeffs`, `pixel_to_geo`, `geo_to_pixel`); EPSG:4326 против
  проекционной CRS обрабатывается в `meters_per_pixel` / `velocity_px_to_metric`. Азимут: 0° = север,
  по часовой стрелке (`az_unit`, `vel_to_az`).

### Форматы входного DEM (`load_dem`, проверяются по порядку)
1. GeoTIFF через `rasterio`; 2. TIFF через `tifffile` + сопутствующий `meta.json`; 3. `.npy`; 4. вложенный
**текстовый пакет** — блок `-----BEGIN BASE64 NPZ-----` (base64-кодированный `.npz` с `dem` + `meta_json`)
внутри обёртки `-----BEGIN ARENA DEM PACKAGE-----`. `dem_package_sibir.txt` — это формат 4.

## Вложенные данные
- `dem_package_sibir.txt` — DEM (Сибирь, EPSG:4326, ~1522×786 пикс.) в текстовом NPZ-формате-пакете.
- `nmea_data_sibir.txt` — сырой NMEA-лог радиовысотомера (`$GPGGA`).
- `heights_m.txt` — извлечённые высоты (по одной на строку), прямой вход для `--heights`.
- `tchk.csv` — эталон (`t_s,x_px,y_px,elevation`), только для отладки.
- `result/` — выходные файлы последнего запуска.
- `Описание первичное.pdf`, `Расширенное описание.pdf` — описание кейса на русском.
